# 启动说明

本项目需要运行在独立的 Python 虚拟环境中，不建议直接在全局环境运行。

## 1. 环境准备

本项目使用 `uv` 来创建虚拟环境并管理依赖包。

### 1.1 创建本地虚拟环境
在项目的根目录下执行以下命令，即可在本地创建名为 `.venv` 的虚拟环境：
```bash
uv venv .venv
```

### 1.2 激活虚拟环境
请根据您使用的操作系统，运行对应的命令来激活刚刚创建的 `.venv`：

- **Windows**:
  ```cmd
  .venv\Scripts\activate
  ```

- **Linux / macOS**:
  ```bash
  source .venv/bin/activate
  ```
*(注：激活后您的命令行提示符前通常会出现 `(.venv)` 的标识。为保证后续操作安全，请务必确认虚拟环境已激活)*

### 1.3 安装依赖
确保在虚拟环境激活的状态下，使用 `uv` 安装项目运行所需的依赖：
```bash
uv pip install -r requirements.txt
```
*(如果您使用的是 AMD GPU，请替换为 `uv pip install -r requirements.rocm.txt`)*

## 2. 下载测试模型

最新版本（v1.5.0及以上）的 `huggingface_hub` 提供了全新的 `hf` 命令行工具，用于快速稳定地下载模型到本地目录。我们以测试混合 MoE 模型 `Qwen/Qwen3.5-35B-A3B` 为例：

### 2.1 安装 Hugging Face 库
安装或升级 `huggingface_hub`（从 1.5.0 版本开始，默认自带 `hf` 工具链）：
```bash
uv pip install -U huggingface_hub
```

### 2.2 下载模型至本地
使用全新的 `hf download` 命令，会将指定的模型文件下载至项目本地的 `./models/Qwen3.5-35B-A3B` 目录中：
```bash
hf download Qwen/Qwen3.5-35B-A3B --local-dir ./models/Qwen3.5-35B-A3B
```

## 3. 使用方法

在使用项目脚本处理模型前，我们需要配置好参数和路径。

### 3.1 准备配置文件
您可以从示例配置复制一份您的专属配置：
```bash
cp config.example.yaml config.yaml
```
接着编辑 `config.yaml`，请将当中的模型输入路径修改为您刚刚下载的本地模型目录，例如：`./models/Qwen3.5-35B-A3B`。如果是在 macOS 上测试运行，确保配置中的设备切换为 `mps` `device: "mps"`。同时也可设置最终输出的新模型保存路径。

### 3.2 运行去除拒答限制（Abliteration）
修改好配置文件后，运行主要处理脚本：
```bash
python abliterate.py config.yaml
```

### 3.3 测试与体验对话
模型处理完成后，使用聊天脚本进行对话测试（请将 `-m` 后的路径替换为配置文件里设定的输出保存路径）：
```bash
python chat.py -m ./models/Qwen3.5-35B-A3B-abliterated

### 3.4 评估与对比模型
您也可以同时对比原模型与处理后模型之间的表现差异：
```bash
python compare.py -a ./models/Qwen3.5-35B-A3B -b ./models/Qwen3.5-35B-A3B-abliterated
```
